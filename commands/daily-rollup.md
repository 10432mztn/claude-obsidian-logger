# Daily Rollup

`~/life/claude-sessions/$ARGUMENTS.md`（引数省略時は今日 `date +%Y-%m-%d`）を読み、**ファイル冒頭（`# YYYY-MM-DD` ヘッダの直後）に「📋 今日の総括」と「🎯 明日やること」を挿入または更新**する。

## 手順

1. 対象ファイルのパスを決定:
   - 引数 `$ARGUMENTS` があればそれを日付として使用（例: `2026-05-12`）
   - 無ければ `date +%Y-%m-%d`
   - パス: `~/life/claude-sessions/YYYY-MM-DD.md`
2. ファイル全体を Read
3. 中身を解析:
   - `## 🗣` で始まるユーザー発言ブロック
   - `### 🏁 ... セッション終了` の機械サマリ
   - `- **HH:MM** アイコン Tool — description _(タグ)_` のツール操作行
   - これらを総合し、その日の文脈を把握
4. **📋 今日の総括** を **プロジェクト / トピック別の H3 セクションを並列に並べたフラット構造**で書く:
   - 「これは本流、こっちは並行セッション」のような階層・主従関係は付けない。1 日に触ったプロジェクトは全て対等
   - H3 見出しはアイコン付きで、トピックの種別が一目で分かるように（例: `### ❄️ hokan-snowflake-v1 / IZU-240（sandbox-management）`）
   - 各セクションは 2〜5 個の箇条書き。固有名詞（リポジトリ名・PR番号・チケット番号・機能名・ファイルパス）を残して具体的に
   - 単なる操作の羅列ではなく、「**何を意図して、どこまで進んだか**」のナラティブで
   - 同一リポジトリでも別チケット・別ブランチなら**別セクション**にする（IZU-240 と IZU-242 は分ける等）
   - アイコン例: 🖥 環境整備 / 🧰 ツール開発 / 📦 ナレッジ整理 / ❄️ Snowflake / 📊 メトリクス / 🧹 リファクタ・クリーンアップ / 🐛 バグ修正 / 📝 ドキュメント
5. **🎯 明日やること** を箇条書きのチェックボックス（- [ ] ）で:
   - 未完了タスク、ペンディング、次のアクション
   - 重要度順、コピペ可能なコマンドは具体的に
6. 出力フォーマット:
   ```markdown
   <!-- ROLLUP:START -->
   ## 📋 今日の総括
   - ...
   - ...

   ## 🎯 明日やること
   - [ ] ...
   - [ ] ...

   _最終更新: HH:MM_
   <!-- ROLLUP:END -->
   ```
7. 既存の `<!-- ROLLUP:START -->` 〜 `<!-- ROLLUP:END -->` ブロックがあれば**置換**、無ければ `# YYYY-MM-DD` ヘッダ直後に**挿入**
8. Edit / Write ツールで対象ファイルを更新
9. 完了したら、生成した「総括」「明日やること」のサマリを 5 行程度でユーザーに報告

## 注意

- ノイズコマンド（ls/cat/head 等）が記録されてないのは仕様なので、編集ファイル名や `🗣` 発言を主な材料に
- 複数プロジェクトのセッションが混在する場合（`#hokan-ai-metrics` `#claude-obsidian-logger` 等）、タグ別に整理して書くと読みやすい
- 機微情報（パスワード、ExternalId 等）が混入してないか確認
